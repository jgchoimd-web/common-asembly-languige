Portable assembly IR

Write one low-level language. Lower it into many assembly worlds.

CommonASM is a common intermediate assembly for language builders who do not want to write a separate backend by hand for every CPU, VM, toy machine, or esoteric target.

Toy and encoding backends CHIP-8, SUBLEQ, Brainfuck, FRACTRAN, Rule 110, and friends.

Target strategy

Three support levels keep the promise honest.

Primary

These targets are the reference outputs for the portable IR design. They are the first place to verify new CommonASM instructions.

  • x86_64-nasm
  • riscv64-gnu

Experimental assembly and IR

These outputs use real assembly or familiar IR style where possible, while staying inside the CommonASM portable subset.

  • i386-nasm, aarch64-gnu, rv32i-gnu
  • mips32-gnu, ppcg4-gnu, sparcv9-gnu
  • m68k, avr, ebpf, wasm, llvm-ir

Encoding and pseudo targets

Machines with very different memory models, tiny register sets, or esoteric semantics use readable pseudo assembly or source encodings.

  • chip8, subleq, redcode
  • brainfuck, fractran, cellular-automaton
  • pdp11, vax, mix, lc3

Language shape

A small assembly vocabulary for compiler backends.

CommonASM has sections, constants, data declarations, virtual registers, typed memory access, arithmetic, branches, function helpers, stack operations, and Linux-style syscall helpers for targets that can lower them.

examples/hello.cas
const stdout = 1

.data
msg: string "Hello from CommonASM\n"

.text
global _start

_start:
  syscall write, stdout, msg, msg_len
  syscall exit, 0

Build and compile

The C AOT compiler is the reference implementation.

1

Build

gcc csrc/commonasmc.c -o build/commonasmc.exe
2

Inspect the CLI

build/commonasmc.exe --help
build/commonasmc.exe --list-targets
3

Compile to a primary target

build/commonasmc.exe examples/hello.cas \
  --target x86_64-nasm \
  -o build/hello_x86.asm
4

Try an experimental target

build/commonasmc.exe examples/vm_ir.cas \
  --target wasm \
  -o build/hello.wat

Developer ergonomics
